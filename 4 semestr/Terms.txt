Атрибут — свойство, которое описывает некоторый аспект объекта. Атрибут можно представить как столбец таблицы сущности.

База данных — сов-сть связанных данных, организованных по определенным правилам, предусматривающим общие принципы описания, хранения и манипулирования независмо от прикладных программ.

Внешний ключ. Столбец одной таблицы, значения в котором совпадают со значениями столбца, являющегося первичным ключом другой таблицы, называется внешним ключом.

Домен — набор допустимых значений для атрибута.

Индекс — объект базы данных, создаваемый с целью повышения производительности поиска данных.

Кардинальное число — кол-во кортежей.

Кардинальность — кол-во уникальных значений в наборе данных.

Коррелирующий подзапрос — подзапрос, в котором обращение идет к внешним строчкам.

Кортеж — строка отношения.

Метаданные — "данные о данных", данные, которые описывают объекты баз данных, а также позволяют упростить способ доступа к ним и управления ими. 
				Метаданные включают определения записей, элементов данных, а также другие данные, представлящие интерес для пользователей или необходимые для работы СУБД.

Нормализация.
	     Нормализация — метод, который используется для определения подходящего набора отношений.
	     Нормализация часто выполняется в виде последовательности тестов с целью проверки соответствия (или несоответствия) некоторого отношения требованиям заданной нормальной формы 
	     и устранения избыточности данных для нормализации реляционной модели до заданной конкретной модели, что позволит предотвратить возможное возникновение анмалий.

Определитель NULL. Значение атрибута в настоящий момент неизвестно или непремлимо для этого кортежа.

Отношение — плоская таблица из строк и столбцов.

Первичный ключ. Потенциальный ключ, который выбран для идентификации кортежей отношений.

Потенциальный ключ. атрибут/набор атрибутов, которые однозначно идентифицируют кортеж отношений.

Представление — виртуальная таблица, которая не хранит в себе данные, а строится на основе запроса к таблицам или другим видам.
		Когда мы пишем SELECT Фамилия, Имя, Отчество FROM Студенты; это и есть представление.

Свойства отношения
		1. Отношение имеет имя, которое отличается от имен всех других отношений в реляционной схеме.
		2. Каждая ячейка отношения содержит только одно элементарное (неделимое) значение.
		3. Каждый атрибут имеет уникальное имя.
		4. Значения атрибута берутся из одного и того же домена.
		5. Каждый кортеж является уникальным, т.е. дубликатов кортежей быть не может.
		6. Порядок следования атрибутов не имеет значения.
		7. Теоретически порядок следования кортежей в отношении не имеет значения. (Но практически этот порядок может существенно повлиять на эффективность доступа к ним.)

Ссылочная целостность — внешний ключ должен ссылаться на строчку другого отношения. 
			внешний ключ должен обязательно ссылаться на потенциальный ключ другой таблицы.

Степень. Степень отношения определяется количеством атрибутов, которое оно содержит.

СУБД — ПО с помощью которого пользователи могут определять, создавать и поддерживать БД.

Суперключ. Атрибут или множество атрибутов, которое единственным образом идентифицирует кортеж данного отношения.

Сущность — таблица, содержащая основные данные, в которой строки представляют элементы, а столбцы — атрибуты.
	В проекте "Организация" м.б. сущности "Работник", "Клиент", "Офис", "Продукция".

Таблица, содержащая внешний ключ, является потомком, а таблица с первичным ключом - предком.

Транзакция — одна операция/набор операций, воспринимаемая БД, как единое целое и переводящая СУБД из одного непротиворечивого состояния в другое.

Триггер — програмная единица, срабатывающая в ответ на действия пользователя.

Хранимые процедуры и функции — это именованные блоки PL/SQL, которые могут принимать параметры и вызываться на выполнение.

Целостность сущности. Ни один из атрибутов первичного ключа не может иметь значение null.
